Общие настройки дополнения



Общие настройки дополнения (доступные для изменения по кнопке "настроить" на вкладке "Расширения" в меню Инструменты->Дополнения)

Disable E10S automatically

Выключить E10S автоматически		 E10S - это технология повышенной изоляции вкладок FireFox, повышающая безопасность пользователей (начиная с версии FireFox 48). Однако, она не совместима с работой функций дополнения. Поэтому E10S нужно отключить, если вы хотите использовать дополнение.

Disallow resource scheme

Запретить схему resource://		 Любой скрипт любой страницы может обратится к некоторым служебным файлам FireFox. Это как файлы самого FireFox, так и его дополнений, которые могут встраиваться дополнением внутрь страницы. По наличию таких файлов и их содержимому пользователь может быть однозначно идентифицирован любым сайтом.
В случае включения (установлен флажок) фильтра, дополнение будет препятствовать попыткам доступа к resource:// . Однако, будет позволять доступ к ресурсам, декларируемым другими дополнениями, чтобы не нарушать их работу.
Включено по умолчанию.
Disallow resource scheme by strong

Строго запретить схему resource://		 Аналогично "Disallow resource scheme". Отличие в том, что будут блокироваться и ресурсы других дополнений, что повлечёт за собой более высокий уровень приватности, но нарушит (возможно, незначительно) работу этих дополнений.
Отключено по умолчанию.
Non block https filter

Неблокирующий https-фильтр		 По умолчанию включено. Фильтр "только HTTPS" будет пытаться перенаправить http-запросы на https вместо их простого блокирования.
Minimum TLS strong for HTTPS filter

Минимальная стойкость шифрования для фильтра "только HTTPS"		 Интервал значений: от 0 до 100.
Фильтр HTTPS отключает на доменах, где он включён, возможность подгрузки нешифрованной информации, исключая возможность ocsp-запросов.
Если протокол подгрузки не https, то он блокирует такую подгрузку. Однако, даже если протокол подгрузки https, это не значит, что соединение зашифровано. Поэтому фильтр HTTPS будет блокировать нешифрованные и недостаточно шифрованные соединения.
Настройка определяет, насколько стойким должно быть соединение, чтобы не быть заблокированным. Например, значение настройки, равное 1, будет говорить, что соединение будет заблокировано, если стойкость шифрования будет оценена не выше 1%. Значение 0 будет блокировать только незащищённые соединения.

У данной фильтрации есть недостаток: блокирование соединения осуществляется уже после отсылки информации на сервер. То есть хотя данные с сервера и не будут обработаны FireFox, куки и другая информация на сервер уже уйдут.
Blocking Log enabled

Включить лог блокировок		 По умолчанию флажок поставлен, что говорит о том, что дополнение будет собирать информацию о сделанных блокировках.
При снятии данного флажка дополнение не будет вести лог, что позволит немного сэкономить память и увеличить быстродействие, однако вкладка "Лог блокировок" панели дополнения будет пуста.
Estimate TLS

Оценка стойкости TLS		 По умолчанию флажок поставлен, что говорит о том, что дополнение будет собирать информацию о стойкости шифрования и оценивать стойкость шифрования сайта.
При снятии данного флажка дополнение не будет собирать эту информацию, что позволит немного сэкономить память и увеличить быстродействие, однако стойкость шифрования оцениваться не будет и вкладка "TLS" будет пуста.
mainpanel.font

Шрифт панели дополнения		 Значение по умолчанию: 16px "Times New Roman" (с кавычками).
Определяет, каким шрифтом отображается информация панели дополнения. На некоторых компьютерах панель дополнения не помещается в экран. В таком случае можно вручную поставить значение величины шрифта меньше. Например, 13px "Times New Roman".
mainpanel.height

Высота панели дополнения		 Значение по умолчанию: 880
Определяет, какой высоты будет панель дополнения. Если высоты не достаточно, то панель должна прокручиваться.
Hosts allowed on start

Разрешать запросы при старте дополнения		 Значение по умолчанию: false
В момент инициализации дополнения, по умолчанию, дополнение блокирует все запросы. Для успешной работы некоторым дополнениям нужно посылать запросы при старте FireFox, когда HUAC ещё не инициализирован. Если это так, хосты, к которым разрешён доступ, нужно добавить в этот список. Разные хосты разделяются пробелами. Имя хоста, начинающееся с точки, разрешает и хост, и его поддомены. Пример:
.services.mozilla.com addons.cdn.mozilla.net download.mozilla.org self-repair.mozilla.org
OCSP allowed on start

Разрешать OCSP-запросы при старте дополнения		 Значение по умолчанию: false
В момент инициализации дополнения, по умолчанию, дополнение блокирует все запросы. Поэтому служебные запросы, в том числе, OCSP-запросы, могут быть заблокированы. Если установлен флажок, то OCSP-запросы при старте дополнения будут разрешены.
debug.writeSettingFilePathes

Записывать в консоль факт успешного открытытия конфигурационных файлов в начале работы дополнения		 Значение по умолчанию: false
При старте дополнения в консоль будут выводится пути открытых конфигурационных файлов.
